# Fundamentos Web

## Reflexión

### ¿Qué fue lo más difícil?
Lo más difícil fue trabajar con Git, especialmente entender cómo funcionan los commits y cómo subir el proyecto a GitHub.

### ¿Qué no entendí completamente?
No entendí completamente cómo Git guarda los cambios internamente ni cómo funcionan las ramas en proyectos grandes.

### ¿Para qué creo que sirve Git en proyectos reales?
Git sirve para llevar el control de los cambios en un proyecto, trabajar en equipo sin perder información y poder volver a versiones anteriores si ocurre algún error.

### ¿Por qué es importante estructurar bien un proyecto?
Es importante porque facilita la organización, el mantenimiento y el trabajo en equipo. Una buena estructura hace que el código sea más claro y profesional.
